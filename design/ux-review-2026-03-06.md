# UX Review — 2026-03-06

Live browser review of all main pages (dashboard, schedules list/detail, issues, projects, assets, spending).
Screenshots in `screenshots/current/`.

---

## HIGH

### Touch targets below 44px on action buttons
**File:** `maintenance/static/css/app.css:63-67`
**Harm:** `.btn-action` buttons are ~28-30px tall. The Done/Edit pairs in the schedules table are frequently mistapped on touchscreen devices.
**Fix:** Increase padding to achieve a 44px minimum height.
**Status:** Fixed

### Misleading empty state when a filter is pre-applied
**Files:** `maintenance/templates/issues/list.html:86`, `projects/list.html:80`, `assets/list.html:90`
**Harm:** Issues defaults to "Active (not resolved)". A new user with no data sees "No issues match your filters" — implying hidden records exist rather than that the list is genuinely empty. Same on Projects and Assets.
**Fix:** Detect when no records exist at all (vs. filtered-out records) and show a distinct onboarding empty state with a call to action linking to the create form.
**Status:** Fixed

### Dark mode preference ignored — `data-theme="light"` hardcoded
**File:** `maintenance/templates/base.html:3`
**Harm:** PicoCSS supports `prefers-color-scheme` automatically, but `data-theme="light"` unconditionally overrides it. Users with OS dark mode get a forced light UI. The dark mode color token overrides in `app.css` already exist but never fire.
**Fix:** Remove `data-theme="light"` from `<html>` to restore system preference detection.
**Status:** Fixed

## MEDIUM

### No feedback during "Mark Done" request on schedule detail
**File:** `maintenance/templates/schedules/detail.html:76-86`
**Harm:** Clicking "Mark Done" gives no immediate visual response — button stays active, page is blank during reload. On slow connections the user will re-click, creating duplicate work logs.
**Fix:** Add `hx-disabled-elt="this"` to the submit button and guard the reload: already has `if(event.detail.successful)` guard, but button needs disabling during the request.
**Status:** Fixed

### No active page indicator in nav
**File:** `maintenance/templates/base.html:46-51`
**Harm:** No nav link is visually marked as current. Screen reader users get no `aria-current="page"` signal. Keyboard users can't tell where they are from the nav.
**Fix:** Use a Django context processor or per-template block to set `aria-current="page"` on the matching nav link, and add a CSS rule to visually distinguish it.
**Status:** Fixed

### Schedules list table unusable at mobile widths
**Files:** `maintenance/templates/schedules/list.html:78`, `maintenance/static/css/app.css:29-40`
**Harm:** At ~400px, the name column wraps every row to 2-3 lines. 40 rows at 3 lines each creates an extremely long page. The Actions column is clipped.
**Fix:** At mobile breakpoint, hide low-value columns (Category, Priority) so Name + Status + Actions remain readable without horizontal scrolling.
**Status:** Fixed

## LOW

### Generic HTMX error message — no recovery guidance
**File:** `maintenance/templates/base.html:79`
**Harm:** "Something went wrong. Please try again." gives no context about what failed or whether data was lost.
**Fix:** Differentiate network errors from server errors in the error handler, and for form submissions, keep input intact.
**Status:** Fixed

### Breadcrumb missing `aria-current="page"` on last item
**File:** `maintenance/templates/schedules/detail.html:10`
**Harm:** Screen readers announce the breadcrumb as a plain list with no indication of current location.
**Fix:** Add `aria-current="page"` to the last `<li>` in all breadcrumb navs.
**Status:** Fixed
